Unrolling Theano Scans for Fun and Profit

Date Thu 25 August 2016 Tags Theano / Scan / Programming / Performance / Python / RNN

TL;DR: Unrolling the built-in loop construct in Theano can lead to a massive speedup under certain circumstances and can be applied e.g. to speedup RNN-training.

Theano is an amazing piece of software. It enables the user to conveniently compose mathematical expressions using a library of pre-defined operations, in particular differentiable multi-variate real-valued functions that typically arise as objective functions in machine-learning methods. Since the goal is then to optimize these functions, expressing computations in Theano has two major benefits:

  • Theano can automatically derive gradients: while specifying the objective function is usually straight-forward, deriving gradients can be a tedious and error-prone task. Since Theano's elementary operations implement partial derivatives, Theano can figure out gradients of compound expressions by applying the chain rule.
  • Theano expressions can be executed on different backends, in particular GPUs that can lead to speedups of 1-2 orders of magnitude

These features make Theano particularly suitable for deep learning, since deep neural networks consist of several layers of similar structure based on simple operations. The training procedure can often be posed as a continuous, unconstrained optimization problem, and training large networks on huge amounts of data require the processing power of GPUs to complete in a reasonable amount of time.

Here, we are particularly interested in recurrent neural networks (RNNs), that are being successfully used to model sequences. RNNs map sequences of inputs to sequences of real-valued hidden vectors, where the computation of such a hidden vector at a time step $t$ depends on the corresponding input at time $t$ as well as on the hidden vector at the previous time step. In order to train an RNN, this recursive formulation is explicitely unrolled over a given input sequence, resulting in a feed-forward network. Since unrolling an RNN leads to identical computations at every time step, it is convenient to use a loop construct to describe it. In Theano by default loops can be expressed using the scan operator. While the scan operator is very convenient and powerful, it appears to introduce overhead that cannot be optimized away. I found that unrolling the loop explicitely can lead to an almost 10-fold speedup in terms of computational performance. Since unrolling the loop generates many more nodes in the computation graph, the compile time increases dramatically. This one-time expense is however easily outweighed by the massive gains during training and can be to an extend mitigated by pickling the compiled Theano function.

This optimization is mentioned here and there, but I did not find a tutorial on this and thought it could be useful.

import theano
import theano.tensor as T
from numpy import *
from scipy.misc import logsumexp
import time

def stepTemplate(xt, hprev, Wh, Wi, mvm, sigma):
    at = mvm(Wh, hprev) + Wi[xt]
    ht = sigma(at)
    return ht

stepT = lambda x, h, Wh, Wi : stepTemplate(x, h, Wh, Wi, T.dot, T.tanh)
step  = lambda x, h, Wh, Wi : stepTemplate(x, h, Wh, Wi, dot, tanh)

def fwd(x, y, h0, Wh, Wi, Wo):
    h = h0
    B,D = len(x), Wh.shape[0]
    H = zeros((B, D)).astype(float32)
    for (t,xt) in enumerate(x):
        h = step(xt, h, Wh, Wi)
        H[t] = h
    Y = dot(H, Wo)
    lp = Y[range(B), y] - logsumexp(Y,1)
    return H, Y, lp

H_np, Y, lp = fwd(x, y, h0, Wh, Wi, Wo)

params = {'Wh':(D,D), 'Wi':(M,D), 'Wo':(D,M)}
tParams = {name : T.matrix(name) for name in params}

def getSharedGrad(params):
    return {n: theano.shared(zeros(params[n], dtype=float32),'sg'+n) for n in params}

def getLoss(ty, th0, tH, tParams, shVars):
    tY = T.dot(tH, tParams['Wo'])
    tP = T.nnet.softmax(tY)
    tLoss = T.mean(T.nnet.categorical_crossentropy(tP, ty))
    tGrad = {name:theano.grad(tLoss, var) for name, var in tParams.iteritems()}
    
    updates = [(shVars[name], tGrad[name]) for name in tParams]
    
    sH = theano.shared(zeros((B, D)).astype(float32))
    updates.append( (sH, tH) )
    inputs = [tx, ty, th0, tParams['Wh'], tParams['Wi'], tParams['Wo']]
    
    loss = theano.function(inputs=inputs, outputs=tLoss, updates=updates)
    return loss, sH
